# 稗玉大地插件设计文档

### 1. 引言
稗玉大地插件旨在将 Minecraft 改造为一款以文明演进为**核心**的 RTS 策略游戏。玩家通过创建**领地**，从原始社会起步，发展成**政权**，通过内政、科技、军事、外交随时间逐步发展至现代社会。**核心**最小单位是“**户**”和“建筑”，每个**户**有职业，居住于民居中并产出资源。建筑由玩家手动搭建后通过插件“录入”成为功能性建筑，后期可解锁“自动建造”功能。

#### 1.1 基本
版本需要 1.21 paper核心
- **依赖**：Citizens、Vault


### 2. 术语表
- **政权**：玩家创建的独立国家、政权。属性有领地、人口、资源、科技等。
- **领地**：政权占据的地理区域，以区块（chunk，16*16方块）为最小单位。
- **核心**：一个方块，作为政权的标志物和管理入口。
- **户**：人口的基本单位，相当于一个家庭，居住于民居中。属性有人口、职业、满意度、资源、经济等属性。
- **开拓者资源**：可用于建立新政权、移民的流动人口。该资源属于玩家，仍属于玩家的开拓者资源需要定时消耗维护费，否则将减少开拓者资源并随机找一户加入（移民资源人跑了）。1点开拓者资源就是1个人，需要从原本的户中减少1个人口。
- **功能建筑**：玩家手动搭建后通过插件“录入”成为的建筑，具备特定功能，如生产、商业、军事等。
- **自生长建筑**：被标记为可由系统自动生成的建筑，只能在自生长区域上生成。目标区域面积需要足够大（目标区域面积大于等于待生成建筑的面积），且不能有其他建筑。
- **自生长区域**：用于生成自生长建筑的区域，只要是草径方块即视为自生长区域。连续判断：一个草径方块（A草径方块）的东南西北任一方向相连的是草径方块（B草径方块），那么这个方块（B）也被视为自生长区域，只有同一层y轴的相连草径才能被认为是同一片自生长区域。
- **PAF币**：Peasure and Fortune 的缩写，是后期通用货币（服务器统一虚拟货币），与金共同作为交易媒介。可以在config文件中更改名字。

### 3. 总体设计
#### 3.1 政权与领地
插件以**政权**为核心实体，由玩家建立，每个玩家只能属于一个**政权**。**核心**方块用于管理**政权**所有功能。
**领地**以**区块**（16×16方块）为最小单位，创建成功后的初始面积可在配置文件中设定（默认为一个区块）。
#### 3.2 建筑
建筑分为六大类：民居、生产、商业、军事、公共建筑。
建筑可通过手动建造后“录入”注册成为**功能建筑**，或通过插件建造系统生成。**自生长建筑**消耗**户**的资源，自动建造是玩家主动消耗资源（当地**领地**仓库）瞬间生成。
#### 3.3 人口
为优化考虑，人口以“**户**”为组，是劳动力、士兵和税收的重要来源。插件采用事件驱动与定时任务相结合的方式运行，支持离线发展，大多数数值均可通过配置文件调整。
#### 3.4 资源

#### 3.5 时代
每个领地的时代有所不同，通过特殊事件、生产力发展等影响，推动时代从原始向上古、封建、工业、现代演进。每个时代解锁新功能，如新建筑、职业、兵种、特殊物品等。
- 初始时代为原始社会，仅能建造最简单的茅草屋和采集工具。
- 上古时代开始，玩家可以建造更复杂的建筑，如木民居、商业、城防、铁质工艺等。进入条件：当地有了第一个铁匠建筑，且当地达到三户。

- 现代时代开始，。进入条件：。

### 4. 详细设计

#### 4.1 创建与初始
- 玩家划定初始**领地**（初始**领地**面积大小为配置文件设定，默认一个区块，领地面积大小最小单位为区块）创建**政权**，初始无市议厅等初始建筑，但有**核心**。
- 每个**政权**拥有的基础指标：人口、资源（食物、木材、石材...）、科技、经济。
- 创建国家需要招募已存在的**户**（说服、购买、攻打下来），成为**开拓者资源**，**领地**拥有了三**户**及以上可以成为国家（自立，从属于其他国家没有户数限制）也可以留在领地**政权**的阶段。
流程：玩家放置**核心** → 获得初始**领地**（资源为空） → 通过招募**开拓者资源**或自然迁入**户**获得人口 → 满足三**户**后可升级为国家。

#### 4.2 内政与发展
**政权管理**  
- **领地**只能属于一名玩家，一个**领地**只能从属于最多一个国家。但**领地**可以由多名玩家共同管理，需要领主设置对应管理员，管理员国籍不受限制。

**建筑系统**  
- 玩家手动建造建筑后，可使用插件“录入”该建筑，用特殊工具左右键建筑立方体区域的对角点后输入指令，使其成为可被系统识别的**功能性建筑**，设定的值（例如民居有**户**口数）需要服务器管理员人工审核，审核通过后才能成为**功能性建筑**。
- 用插件建造系统（非手动搭建，不包含第一次手动建筑的时候）生成过3次及以上可以被系统自动生成，被标记为**自生长建筑**。
- 使用插件建造系统来建造，需要消耗对应的资源，为优化考虑，大多数建筑瞬间生成。

**资源循环**  
- 农民生产作物，磨坊、食品工场将作物产出食物。矿工开采矿石，工匠将原料转化为工具和武器。

**科技树**  
- 通过建造科研建筑（如图书馆、学院、研究院）解锁新技术，推动时代从原始向古典、封建、工业、现代演进。每个时代解锁新功能。

#### 4.3 人口、户、职业
**人口机制**  
- 对**政权**来说不会自带人口，但民居会带**户**，每**户**自带一名人口。每**户**提供人口上限（4~6随机），人口随时间自然增长（受食物盈余影响）。
- **初始人口**：整个世界最开始的人口是由管理员手动创建民居自带的**户**，但此**户**不属于任何国家、**领地**。人口是劳动力、士兵和税收的来源。
- **人口死亡**：战争、饥荒、疾病会减少人口，自然死亡（老死）以家庭为单位，定时减少一定人数，现实2~3天。
- **人口出生**：每一户自带计时器，现实1.5天一轮，新增人口1~2人。
- **人口迁移**：玩家可以消耗**开拓者资源**以达到移民需求；默认不用消耗**开拓者资源**（即民居不带初始户口），需要等待新户口慢慢迁入这里，新户口迁入时，需要减少原本**户**口的人数。

**职业分化**  
- 人口可按户分配，大致分为农民、矿工、工匠、学者、商人、职业士兵等职业，不同职业居住对应建筑，产出不同资源。
- 为优化考虑，职业以一户（家庭）作为最小单位，一户世代同职业，铁匠生铁匠，农民生农民。职业会随机变化，但几率极低，且职业变化会减少户的收入经济；玩家可以付钱给户要求转职。

**户机制**  
- 户基于民居，民居资源足够时（在户的人口数接近户的人口上限数时几率较大），会建新建筑、升级原本建筑，新的建筑会带来新的户，并继承原先的户（新的户带的人口需要减少原先的户对应人口）。
- 户也有自己的值，人口数、资源量（食物）、经济。食物为0后每30分钟减少1~3人。

#### 4.4 社会与经济
**交易与货币**  
- 早期以物易物，然后用金作为货币，后期使用**PAF币**与金。为优化、简化考虑，不以服务器层面提供各国自己的货币，统一使用金或**PAF币**。

**税收与财政**  
- 统治者设定税收，人口定期缴纳**PAF币**。**PAF币**用于维护领土（硬需求）、维持国防治安、购买特殊道具、提高领地等级或公共领域、建筑。

**居民满意度**  
- 受税率、经济、食物供应、战争影响。满意度过低可能引发叛乱率提高、生产效率下降、犯罪率等问题。
- 叛乱分两种，一种只是不上交税、不服役，且攻击政府人员（士兵、玩家）。另一种严重的是集体变成一个独立的领地，玩家需要重新攻占回来（但城堡对玩家绝对忠诚，除非被其他玩家控制住）。每一户可视。

**运输**  
- 为优化、简化考虑，运输只需要消耗对应经济、等待一定时间即可，不需要有实体商队，若周围有运输站、驿站、港口则可以减少运输所需要的时间。

#### 4.5 军事与战争
**军队组建**  
- 战时职业士兵户可直接投入战斗，其他可从可征召职业户（学者、工匠职业的户不可被征召）人口中征召士兵，可以点政策（休农期农民要军事训练，提高战斗力）；平时士兵为职业士兵，整户无产出但是视为职业士兵，可在兵营中训练，战斗力更高。士兵分为多种兵种（如步兵、远程兵种、骑兵，后期现代化兵种）。

**防御设施**  
- 建造城墙、塔楼、陷阱增强领地防御，驻军可主动反击入侵。

**战争机制**  
- **士兵死亡**：士兵死亡时随机从相对应**户**中减少1名人口，职业士兵则减少职业士兵**户**的，征召的士兵则随机一**户**（**户**的职业必须是可以上战场的）。
- **士气**：宣战需消耗资源或拥有外交理由。无理由宣战会降低士气，有理由、资源充足则提高士气。
- **装备**：士兵需要的装备需要从工匠铺生产，否则战斗力低。
- **战斗方式**：玩家作为将领，参与并指挥AI士兵作战；此时玩家能力将大大削弱（为平衡考虑），每10s只能放置一个方块，且方块不能悬空（即~ ~-1 ~不为air）。玩家带领AI士兵跟随玩家，士兵自动攻击附近敌人。玩家用领导权杖操控士兵前往某位置待命或进攻。
- **攻防**：防御设施可以被摧毁，城墙用投石机、大炮可以破坏个洞之类的，城门可以打坏，在围城过程中攻城方可以填上护城河。攻城（仅NPC）可以架设云梯，可以上城墙开城门，守方用斧子推下云梯。
云梯：攻城ai士兵两个，一前一后头顶水平横着的梯子（实体动画），动画中（运输时）的长度为5格，攻城ai士兵到达城墙下时，动画梯子消失，自动放置在城墙上，高度为城墙高度。
护城河：围城阶段，每过一定时间在护城河区域填一个方块，直到填完或者在攻击时暂停。
- **结束**：击退敌方后可以占领领土，插占领旗帜（各个区块都需要一个旗帜，这个区块被标记为xx方待占领区域，区块内不能有敌军士兵），并达到一定时间后，即可征服该区块，占领区块足够多，则迫使其成为自己的领土，或进行停战协议。

**战争胜利**  
- 任满足其一即胜利：打到对方投降，打到插的占领旗帜到达时间（按计算公式），打到对面士兵全没了。最后都没达成的话，设定时间（现实7天或配置文件自定义）自动结束战争。

#### 4.6 外交与扩张
- **外交关系**：政权间可闭关、结盟、宣战、正常贸易、签署条约。（除闭关外，仅玩家层面自己整）
- **领土扩张**：通过占领荒地或侵略其他国家领土，所有占领的土地都要有维护费，未缴费则有可能重回荒地或无法有效管理（失去管理权限）。
- **持续发展**：玩家可随时上线调整政策、应对随机事件（如天灾、叛乱），离线时文明按设定自动运行。

#### 4.7 建筑分类细化
建筑大致分为“民居、生产、商业、军事、公共建筑”。

**民居**  
- 纯民居、商业居住混合。分低密度、中密度、高密度（后期）。
- 值有：建筑等级、建筑类型、户口数、户口上限数、满意度、水资源、[附带商业]
  - 低密度：1~3户，独户小院子、大宅
  - 中密度：3~12户，公寓、筒子楼
  - 高密度：12~100户，塔楼、大型公寓

**生产**  
- 基本每个（带*号除外）都有对应的辅助配套工作间，例如农田工作间（原型：放工具杂物；效果：提高农产品生产效率）。
- 值有：建筑等级、建筑类型、职位数（可招聘的人数）、产品、效率（产量）
  - **农业**：水井、农场、牧场、渔场、养殖渔场、灯塔*、林场。
  - **工业**：
    - 前期：
      - 建筑业：瓦场、陶瓷场、砖场
      - 林业：伐木场、烧炭场、木匠铺*
      - 五金业：矿场、冶炼厂、工坊*、铁匠铺*
    - 后期：暂略

**商业**  
- 分为两大类，一类为npc、政府、系统服务（非玩家性质）；一类可以为玩家服务，可以出售给玩家物品或提供全球市场（玩家性质）。
- 值有：建筑等级、建筑类型
  - 非玩家性质：
  - 玩家性质：同上，但是可以由领主、管理员设定物品出售

**军事**  
- 暂略。值有：建筑等级、建筑类型

**公共建筑**  
- 值有：建筑等级、建筑类型
  - 教育：学校、图书馆、研究所、大学
  - 政府：市政厅、法院、监狱
  - 医疗：医院、诊所
  - 宗教：神庙、教堂、寺庙
  - 娱乐：酒馆、剧院
  - 交通：道路、港口、运输站、驿站

#### 4.8.5 接口设计

**命令**  

| 命令 | 权限 | 说明 |
|------|------|------|
| /baiyu create | baiyu.admin | 创建政权（需手持核心物品） |
| /baiyu register <类型> <子类> | baiyu.builder | 录入建筑 |
| /baiyu build <建筑类型> | baiyu.builder | 自动建造（消耗资源） |
| /baiyu core | baiyu.user | 打开核心管理界面 |
| /baiyu tax <税率> | baiyu.leader | 设置税率 |
| /baiyu recruit | baiyu.leader | 招募开拓者 |
| /baiyu declare <目标> | baiyu.leader | 宣战 |
| /baiyu staff | baiyu.user | 获取领导权杖（战争用） |

**特殊物品**  
- **核心物品**：用于创建政权的物品，右键放置生成核心方块。
- **建筑录入工具**：木棍，左键/右键选取对角点。
- **领导权杖**：战争期间用于指挥AI士兵，右键打开悬浮菜单（可指令待命、进攻、撤退）。

**事件**  
- 建筑录入事件
- 人口出生/死亡事件
- 战争开始/结束事件
- 叛乱事件
